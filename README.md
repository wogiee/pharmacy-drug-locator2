# 약품 위치 찾기 웹앱

약국 선반 위치를 빠르게 찾고, 관리자가 약품별 위치와 상세설명을 수정할 수 있는 첫 버전입니다.

## 현재 기능

- CSV 기반 약품 목록 로드
- 약품명, 공식 제품명, 카테고리, 제조사, 설명 통합 검색
- 위치 필터: `1-L`부터 `15-R`, `PS`, `DR`
- 카테고리에 `파스`가 있으면 위치를 자동으로 `PS` 처리
- 카테고리에 `드링크`가 있으면 위치를 자동으로 `DR` 처리
- 관리자 수정 화면
- 이미지 URL, 출처 URL, 상세설명 입력
- 브라우저 저장소 기반 저장
- JSON 내보내기/가져오기

## 공식 데이터 연동 방향

공식 제품명, 제조사/수입사, 효능, 사용법, 주의사항, 상호작용, 부작용, 보관법, 이미지 정보는 공공데이터포털의 식품의약품안전처 API를 우선 사용합니다.

- 의약품개요정보(e약은요): 일반의약품 상세설명 및 `itemImage`
- 의약품 제품 허가정보: 공식 제품명, 업체명, 허가정보
- 의약품 낱알식별 정보: 알약 식별/이미지 보조
- 약학정보원 API Center: 별도 가입/승인 후 보조 출처로 검토

공공데이터포털 API는 서비스키가 필요합니다. 서비스키를 받으면 서버 쪽 보강 스크립트로 305개 약품명을 공식명과 매칭하고, 매칭 신뢰도가 낮은 항목만 사람이 검수하는 방식이 좋습니다.

## 서버 공유 버전 제안

5명 정도가 실시간으로 공유하려면 다음 구조가 적합합니다.

- Frontend: 현재 정적 웹앱 유지
- Auth: 관리자 PIN 또는 계정 로그인
- Realtime DB: Supabase 또는 Firebase
- Tables:
  - `products`: 약품 기본정보, 위치, 가격, 재고
  - `drug_profiles`: 공식명, 제조사, 설명, 이미지, 출처, 매칭 상태
  - `audit_logs`: 누가 언제 위치/설명을 바꿨는지 기록

실제 약국 운영에서는 위치 수정 이력이 꽤 중요해서 `audit_logs`를 처음부터 넣는 편을 추천합니다.
