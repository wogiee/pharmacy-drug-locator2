# Supabase 공유 버전 설정

## 1. SQL 실행

Supabase 대시보드에서 `SQL Editor`를 열고, 프로젝트 폴더의 `supabase-schema.sql` 내용을 그대로 실행합니다.

이 SQL은 다음을 만듭니다.

- `products` 약품 테이블
- `audit_logs` 수정 이력 테이블
- 로그인 사용자만 읽기/수정 가능한 RLS 정책
- `drug-images` 이미지 저장소
- 이미지 업로드 정책

## 2. 직원 계정 만들기

Supabase 대시보드에서 `Authentication` -> `Users`로 이동합니다.

직원 5명의 이메일 계정을 만듭니다. 비밀번호는 직원에게 따로 전달합니다.

## 3. 앱에서 로그인

앱을 새로고침하면 로그인 화면이 나옵니다.

Supabase Auth에 등록한 직원 이메일/비밀번호로 로그인합니다.

## 4. 초기 데이터 등록

처음에는 데이터베이스가 비어 있으므로 로그인 후 상단의 `초기 데이터 등록` 버튼을 누릅니다.

그러면 `data/seed-products.json`의 305개 약품이 Supabase `products` 테이블에 등록됩니다.

## 5. 사용 방식

- 약품 검색/카테고리 필터는 모든 로그인 사용자가 사용합니다.
- 약품을 클릭하면 설명이 펼쳐집니다.
- 오른쪽 수정 패널에서 위치, 재고, 가격, 제조사, 사진을 수정합니다.
- 사진 파일을 업로드하면 Supabase Storage `drug-images`에 저장됩니다.
- 다른 기기에서는 약 12초마다 데이터가 자동 동기화됩니다.

## 6. 배포

Vercel 또는 Netlify에 이 폴더를 정적 사이트로 배포하면 됩니다.

빌드 명령은 필요 없습니다.

출력 폴더도 루트 그대로 사용하면 됩니다.
